# Day 39 – CI/CD Concepts

##  Task 1: The Problem

### 1. What can go wrong?

When multiple developers push code and deploy manually:

* Code conflicts and overwrites
* Bugs reaching production
* Broken deployments
* Inconsistent environments
* No rollback mechanism
* Human errors during deployment

---

### 2. What does "It works on my machine" mean?

It means the code runs fine on a developer’s local setup but fails in another environment.

**Why this happens:**

* Different OS or dependencies
* Missing environment variables
* Different library versions
* Local-only configurations

This is a real problem because production ≠ developer machine.

---

### 3. Manual deployment frequency

* Safe deployments: **1–2 times per day (max)**
* Beyond that → high risk of:

  * Mistakes
  * Downtime
  * Unstable releases

---

## Task 2: CI vs CD

### 1. Continuous Integration (CI)

Developers frequently merge code into a shared repository. Each change triggers automated builds and tests.

Catches:

* Syntax errors
* Broken code
* Integration issues

**Example:**
A developer pushes code → tests run automatically → failure stops bad code.

---

### 2. Continuous Delivery (CD)

Code is automatically built and tested, and kept ready for deployment. Deployment is still manual.

“Delivery” = ready to release anytime.

**Example:**
After CI passes → app is deployed to staging → team clicks "Deploy" for production.

---

### 3. Continuous Deployment (CD)

Every successful change is automatically deployed to production without human intervention.

Fully automated release pipeline.

**Example:**
Push → test → build → deploy to production automatically.

---

## Task 3: Pipeline Anatomy

### Trigger

What starts the pipeline
Examples:

* Git push
* Pull request
* Schedule (cron)

---

### Stage

A logical phase in pipeline
Examples:

* Build
* Test
* Deploy

---

### Job

A group of steps executed together
Example:

* Run tests job
* Build Docker image job

---

### Step

A single command or action
Example:

* `pip install -r requirements.txt`
* `pytest`

---

### Runner

The machine that executes jobs
Example:

* GitHub-hosted runner
* Self-hosted server

---

### Artifact

Output generated by pipeline
Examples:

* Compiled app
* Docker image
* Logs

---
